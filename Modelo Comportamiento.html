    DICCIONARIO DE DATOS

    1. Clientes

    Descripción: Información de los clientes del negocio.

    Atributos:
    • ID_Cliente (PK): Identificador único del cliente (Número entero).
    • Nombre: Nombre completo del cliente (Texto).
    • Dirección: Dirección de residencia del cliente (Texto).
    • Teléfono: Número de teléfono del cliente (Texto).
    • Correo: Dirección de correo electrónico del cliente (Texto).
    • Estado: Estado del cliente (activo/inactivo) (Texto).

    2. Productos

    Descripción: Información de los productos disponibles para la venta.

    Atributos:
    • ID_Producto (PK): Identificador único del producto (Número entero).
    • Nombre: Nombre del producto (Texto).
    • Categoría: Categoría del producto (Texto).
    • Precio: Precio unitario del producto (Decimal).
    • Stock: Cantidad disponible del producto en el inventario (Número entero).
    • Estado: Estado del producto (activo/inactivo) (Texto).

    3. Proveedores

    Descripción: Información de los proveedores de productos.

    Atributos:
    • ID_Proveedor (PK): Identificador único del proveedor (Número entero).
    • Nombre: Nombre del proveedor (Texto).
    • Teléfono: Número de teléfono del proveedor (Texto).
    • Correo: Correo electrónico del proveedor (Texto).
    • Estado: Estado del proveedor (activo/inactivo) (Texto).

    4. Ventas

    Descripción: Registro de las ventas realizadas en el negocio.

    Atributos:
    • ID_Venta (PK): Identificador único de la venta (Número entero).
    • Fecha: Fecha y hora en que se realiza la venta (Fecha/Hora).
    • Total: Total de la venta (Decimal).
    • ID_Cliente (FK): Relación con el cliente que realizó la compra (Número entero).
    • ID_Usuario (FK): Relación con el usuario que procesó la venta (Número entero).

    5. Facturación

    Descripción: Información relacionada con la facturación de una venta.

    Atributos:
    • ID_Factura (PK): Identificador único de la factura (Número entero).
    • ID_Venta (FK): Relación con la venta asociada a la factura (Número entero).
    • Fecha_Emisión: Fecha en que se emite la factura (Fecha).
    • Total: Total de la factura (Decimal).
    • Impuestos: Impuestos aplicados a la factura (Decimal).
    • Estado: Estado de la factura (pagada/no pagada) (Texto).

    6. Pagos

    Descripción: Información sobre los pagos realizados para las facturas.

    Atributos:
    • ID_Pago (PK): Identificador único del pago (Número entero).
    • ID_Factura (FK): Relación con la factura que se paga (Número entero).
    • Fecha: Fecha en que se realiza el pago (Fecha).
    • Monto: Monto pagado (Decimal).
    • Método_Pago: Método de pago utilizado (Texto).
    • Estado: Estado del pago (completado, pendiente, fallido) (Texto).

    7. Usuarios

    Descripción: Información de los empleados del negocio que gestionan las ventas.

    Atributos:
    • ID_Usuario (PK): Identificador único del usuario (Número entero).
    • Nombre: Nombre completo del usuario (Texto).
    • Rol: Rol del usuario (Texto).
    • Contraseña: Contraseña para acceder al sistema (Texto).

    Relaciones

    • Clientes (1:N) Ventas: Un cliente puede realizar muchas ventas, pero cada venta está asociada a un solo cliente.
    • Ventas (M:N) Productos: Una venta puede incluir muchos productos, y un producto puede estar en muchas ventas.
    • Proveedores (1:N) Productos: Un proveedor puede suministrar muchos productos, pero un producto proviene de un solo proveedor.
    • Ventas (1:1) Facturación: Cada venta genera una factura única.
    • Facturación (1:N) Pagos: Una factura puede tener varios pagos asociados.
    • Usuarios (1:N) Ventas: Un usuario puede gestionar muchas ventas, pero cada venta es gestionada por un solo usuario.
